STEP 1 — 基礎知識

Webサイトは3つの材料でできてる

🧱

HTML

骨格・構造

「ここにタイトル」「ここにボタン」という部品の配置を決める。家で言えば「間取り図」。

🎨

CSS

見た目・デザイン

色・フォント・サイズ・アニメーションを決める。家で言えば「内装・壁紙」。

⚡

JavaScript

動き・機能

「ボタンを押したら〇〇する」という動作を書く。家で言えば「電気・ドア」。

💡 playceはこの3つすべてが index.html ひとつのファイルに入っています！

🗄️ Supabaseって何？

データを保存する「クラウド上の倉庫」。PostgreSQLというデータベースをコード不要でサクッと使えるサービス。Googleスプレッドシートのめちゃ高機能版。

📡 なぜ必要？

HTMLファイルだけだとデータを保存できない。誰かが「募集を投稿」しても、ページを閉じたら消えてしまう。Supabaseに繋ぐことでみんなの投稿が永続保存される。

STEP 4 — 参加機能の追加

「参加する」ボタンができた理由

コミット：「feat: 募集への参加機能を追加 (join_requests)」

📋 join_requestsとは？

Supabaseの中に作った「テーブル（表）」の名前。誰かが「参加する」を押したとき、誰が・どの募集に・いつ参加したかをこの表に記録する仕組み。Excelで言えば1枚のシート。

⚡ ボタンを押したとき何が起きる？

  • JavaScriptが動く
  • Supabaseに「参加データ」を送信
  • 参加者数のカウントが増える
  • Discord IDを表示する
🖱️ 参加ボタンクリック

⚡ JavaScript起動
→
📡 Supabaseへ送信
→
✅ 画面を更新
STEP 5 — バグ修正 3連発

バグと戦った3つの修正

  • 1
    タグ表示・文字化けバグを完全修正日本語が「????」のような記号に変わる問題。文字コード（UTF-8）の設定ミスが原因でした。
  • 2
    openJ構文エラーを修正JavaScript の書き方ミス（構文エラー）。カッコや引用符の閉じ忘れなどで動かなくなる。コードの「誤字」みたいなもの。
  • 3
    SupabaseのAPIキーを新形式に更新APIキー＝「Supabaseへの入館証」。古い形式から新しい「Publishable key」形式に変更。Supabase側がセキュリティ向上のため仕様変更した。

STEP 7 — フィルター機能

絞り込み検索と雰囲気タグを追加

コミット：「feat: 雰囲気タグ・フィルター・残り時間・Discord IDコピー機能追加」

🏷️ 雰囲気タグとは？

募集に「わいわい」「ガチ」「初心者OK」などラベルを付ける機能。HTMLのdata属性とCSSクラスを使って色分けされたバッジとして表示。

ガチ わいわい 初心者OK

🔍 フィルター機能

「ガチだけ表示したい」など絞り込む機能。JavaScriptが各カードの属性を読んで、条件に合わないカードを非表示にする仕組み。position:stickyでフィルターバーが画面上部に固定される。

Discord IDコピー機能 → ボタンを押すと navigator.clipboard.writeText() というJSの命令でコピーされます。

STEP 8 — モックデータ

テスト用のダミーデータ10件を追加

コミット：「feat: モックデータ10件・フィルター sticky・ヒーローパルス追加」

🧪 モックデータとは？

「モック（mock）＝偽物・テスト用」という意味。実際にSupabaseからデータを取ってくる前に、見た目確認用の仮データをコードに直書きしたもの。「誰も使ってないのにサイトが空っぽに見える」問題を解決。

✨ ヒーローパルスとは？

CSSアニメーションの一種。要素が「ふわっ…ふわっ…」と光って点滅するエフェクト。@keyframes という命令で作り、「注目させたい箇所」に付ける演出。

/* ヒーローパルス */
@keyframes pulse {
  0% { box-shadow: 0 0 0px #ff4655; }
  50% { box-shadow: 0 0 30px #ff4655; }
  100%{ box-shadow: 0 0 0px #ff4655; }
}

BONUS — どうやって公開してるの？

GitHubが自動でデプロイしてくれる

👨‍💻 コードを編集・保存
→
📤 GitHubにコミット（保存）
→
🤖 GitHub Actionsが自動起動
→
🌐 GitHub PagesでURLが更新

📦 コミットとは？

変更を「確定・保存」すること。ゲームのセーブポイントみたいなもの。いつ・どんな変更をしたか履歴が残り、いつでも以前の状態に戻せる。

🤖 GitHub Actionsとは？

コミットを検知して自動で作業をこなすロボット。「コミットが来たら自動でサイトを公開する」というルールを設定しておくと、ワンクリックで反映される。
